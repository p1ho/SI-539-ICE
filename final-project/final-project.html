Engineers Evaluation

Site: https://campusinfo.umich.edu/

Findings

Use the correct HTML element for your content

Fail

  • The source code looks it might autogenerated, thus it is very malformed. Running it through validator shows lots of errors. Running it through w3c validator shows lots of errors in xmlns attributes in the html tag.
  • In some of the <ul> lists, there is a '|'corrector in between <li> tags; no text content should be allowed between <li> tags within a <ul> tag.
  • It looks like some <span> tags in the page actually contain way more elements than what is usually expected of the <span> tag. A <div> or even a <section> would make more sense.
  • Some orphan tags exist (like an </a> without an <a>)

Support keyboard navigation

Fail

  • While there is tabbing between different html elements, some functionalities do not work. For example, hover menu does not expand when it is focused.
  • The FAQ does not support keyboard navigation on this page https://campusinfo.umich.edu/get-questions-answered
  • The map page (https://maps.studentlife.umich.edu/) does not work with keyboard navigation

Understand and use HTML landmarks

Pass

  • For the most, the html structure is easy to read. It also utilizes html 5 semantic elements such as <section> and <article> to divide the page.
  • It also utilizes the h1-h6 tags pretty effectively, there is no jump (e.g., skipping h3 to h3)

Write good alt text for your images

Pass

  • All img tags have an alt attribute.
  • The alt attributes are meaningful.

Design focus states to help users navigate and understand where they are

Fail

  • There are definitely places where the focus effect is clear when the element is tabbed on (nav bar), but at least half the time it is not clear which element is actually in focus.

Help users understand inputs, and help them avoid and correct mistakes

Fail

  • Even though this is mainly an informational website with minimal form elements, they still exist; and there does not seem to be any measure to help users avoid input mistakes with these form elements (e.g., search bar, map search, Chat with CIC Staff, etc.)

Use ARIA attributes when applicable

Pass

  • Like mentioned before, this website does not have many form elements, but it does include aria tags wherever applicable.

Give users a way to skip top level navigation to access main content

Fail

  • When tabbing it does bring out a visible section that has a link that reads "skip to content", but the link is not navigable through keyboard, it has to be clicked by mouse which kind of defeats the purpose.

Make links descriptive

Pass

  • The links are all descriptive, no "click here" or other ambiguous wording spotted.

Avoid images and iconography in pseudo-elements

Pass

  • The image and icons are all written in html, not through css styling. Well done.

Make SVGs accessible to assistive technology

Pass

  • No SVG spotted, thus not applicable.

Hide decorative elements from screen readers

Fail

  • Although all imgs have alt tags that are descriptive of picture content, some of then should be hidden when using a screen reader.
  • For example, for 'Campus Map, 'Victor Views Blog', 'Happening@Michigan', the background graphics should be hidden to screen readers in my opinion.

Create alternate routes for users to access information

Pass

  • The site seems well optimized for screen reader for the most part.
  • The campus map does not work as well with screen reader, but beacuse it's using Google Map to power it, so it may be quite difficult to accomplish an accessible experience for that.

Links should be visually identifiable and have clear :focus and :active states

Fail

  • Links all have :focus/:hover styling, but no :active styling to distinguish it from focus.
  • However, not styling active seems to be a common practice, so I'm not sure if this is an actual issue, just making a note here.

HTML document should have a language attribute

Fail

  • Most pages have language attributes on the html tag, but it is not there for campus map (https://maps.studentlife.umich.edu/).
  • Should be a simple fix, just make sure the lang attributes are present on all pages.

Recommendations

  1. I suggest finding a way to make the hover menu expand when tabbed on, and make the options inside the menu navigable. This will make the site more navigable by keyboard, and it will tell the screen reader what are the items between those menu items.
  2. If users are using tab to navigate the page, chances are they want to know where they are more than how the page looks, try to be more obvious with the outlines or styling of focused elements. As it stands, about 50% of the links are not very conspicuous when it comes to :focus styling; perhaps making the blue borders a bit thicker or turn it into a different color for instance
  3. Find a way to make the "skip to content" tabbable, and maybe include more meaningful sections than just "content". Currently, not only is the "skip to content" not accessible by keyboard, it is not very informative about the different sections on the page.
  4. The site obviously has awareness for optimizing for screen reader or general accessibility, but just make sure all of the decisions made for accessibility are well informed and tested with potential target audiences.
